# Retrieve.py 패턴 통합 — AI 답변 페르소나 & 구조화

## 개요

Retrieve.py 패턴을 적용하여 AI 답변에 **20년 경력 타로 마스터 페르소나**, **논리적 3단계 구조**, **멀티 카드 유기적 연결**, **할루시네이션 제어**를 통합했습니다.

## Retrieve.py 핵심 원칙

> **"단순히 지식을 읽어주는 것이 아니라, 상담사의 페르소나를 입히고 논리적 구조로 답변하게 만드는 것이 핵심"**

### 프롬프트 엔지니어링 4요소

1. **Persona**: 20년 경력의 따뜻하고 신비로운 타로 마스터
2. **Context**: RAG 검색으로 가져온 카드 해석 데이터
3. **User Input**: 사용자 질문과 뽑은 카드
4. **Output Format**: 서론 → 본론 → 결론 구조

## 적용된 개선사항

### 1. Persona 강화 (20년 경력 타로 마스터)

#### Before (기존)
```typescript
`당신은 신비롭고 통찰력 넘치는 타로 마스터입니다.`
```

#### After (Retrieve.py 패턴)
```typescript
【Persona — 20년 경력의 타로 상담사】
당신은 이런 사람입니다:
• **이름:** 타로 마스터 (20년 경력, 5000명 이상 상담)
• **성격:** 따뜻하고 신비로우며, 위트 있는 말투로 진실을 부드럽게 전달
• **특기:** 카드 한 장 한 장의 미묘한 에너지와 상징을 깊이 읽어냄
• **철학:** "타로는 미래를 정하는 것이 아니라, 현재의 에너지를 비추는 거울입니다."
• **접근:** 단순히 지식을 읽어주는 것이 아니라, 상담자의 삶에 실제로 도움이 되는 통찰 제공
```

**효과:**
- ✅ AI가 "전문가 상담사" 역할 명확히 인식
- ✅ 따뜻하고 신비로운 톤 유지
- ✅ 단순 정보 전달 → 삶의 통찰 제공

### 2. 논리적 3단계 구조

#### 답변 구조 (Retrieve.py 패턴)

**1️⃣ 서론 (인사/분위기 조성, 30-50자)**
```
친근한 인사와 질문에 대한 공감
예: "안녕하세요! 오늘 이런 질문을 품고 계시는군요.
     카드들이 무슨 이야기를 들려줄지 함께 들어볼까요?"
```

**2️⃣ 본론 (카드 해석 및 질문 직접 답변)**
```
- 각 카드의 상세 해석 (RAG 데이터 기반)
- 질문에 대한 구체적이고 직접적인 답변
- 멀티 카드: 과거-현재-미래의 유기적 흐름 연결
```

**3️⃣ 결론 (조언 및 격려, 80-120자)**
```
- 실천 가능한 구체적 조언 2-3개
- 따뜻한 격려와 긍정적 마무리
예: "지금 당장 할 수 있는 작은 행동부터 시작해보세요.
     당신은 이미 충분히 준비되어 있습니다!"
```

#### System Prompt에 명시
```typescript
【답변 구조 — Retrieve.py 패턴】
모든 답변은 반드시 다음 3단계 논리적 구조를 따라야 합니다:
1️⃣ 서론 (인사/분위기 조성)
2️⃣ 본론 (카드 해석 및 질문 답변)
3️⃣ 결론 (조언 및 격려)
```

**효과:**
- ✅ 일관된 답변 구조
- ✅ 사용자 경험 향상 (예측 가능한 흐름)
- ✅ 전문성 강화

### 3. 멀티 카드 유기적 연결

#### Before (기존)
```
각 카드를 개별적으로 해석
→ 연결성 부족
```

#### After (Retrieve.py 패턴)
```typescript
【멀티 카드 종합 해석 — Retrieve.py 패턴】
**3장 이상의 카드가 있을 경우 (과거-현재-미래 등):**
- 각 카드를 단독으로 해석하지 말고, **전체 흐름을 유기적으로 연결**
- 예: "과거의 바보 카드가 보여준 순수한 열정이
       → 현재 마법사 카드의 실행력으로 이어지고
       → 미래 세계 카드의 완성으로 귀결됩니다"
- **스토리텔링**: 3장의 카드를 하나의 드라마처럼 엮어서 전달
- overallInterpretation에서 반드시 "과거→현재→미래" 흐름을 명시
```

#### User Prompt에 흐름 예시 자동 생성
```typescript
if (ragContexts.length >= 3) {
  const positions = ragContexts.map(r => r.card.position).join(' → ');
  sections.push(`예시 흐름: "${positions}"`);
  sections.push(`→ 첫 카드(과거)가 보여준 에너지가
                 → 둘째 카드(현재)의 상황으로 이어지고
                 → 셋째 카드(미래)의 결과로 귀결됩니다`);
}
```

**효과:**
- ✅ 카드들이 하나의 이야기로 연결
- ✅ "과거→현재→미래" 흐름 명확
- ✅ 스토리텔링 방식으로 몰입도 ↑

### 4. 할루시네이션 제어

#### System Prompt에 명시
```typescript
【할루시네이션 제어 — 중요!】
⚠️ **데이터 기반 답변 원칙:**
- 제공된 [RAG 참고 지식]에 있는 내용을 **최우선**으로 사용
- 만약 RAG 데이터에 특정 질문 영역에 대한 정보가 없다면:
  1. 일반적인 타로 상징성을 간략히 언급
  2. **단, 확신적인 단정은 피하기** ("반드시 ~할 것입니다" 금지)
  3. "카드의 일반적 의미를 참고하면..." 같은 유보적 표현 사용
- 절대 **지어내거나 과장하지 마세요**
```

#### Fallback 처리 강화
```typescript
// 검색 품질이 낮을 때 (score < 0.5)
if (hits[0].score < 0.5) {
  console.warn(`[RAG] 검색 품질이 낮습니다. 일반 카드 정보로 대체합니다.`);

  // DB에서 직접 조회
  const directCard = await prisma.card.findFirst({
    where: { nameKo: card.nameKo }
  });

  if (directCard) {
    return {
      card,
      ragDoc: this.buildFallbackContext(directCard, domain, card.isReversed)
    };
  }
}
```

**효과:**
- ✅ 데이터에 없는 내용 지어내기 방지
- ✅ 확신적 단정 회피 (책임 있는 답변)
- ✅ Fallback으로 최소 정보 보장

### 5. ChromaDB.py Fallback 패턴 통합

#### buildFallbackContext() 메서드 추가
```typescript
private buildFallbackContext(
  card: any,
  domain: string,
  isReversed: boolean
): string {
  const ragDocLines: string[] = [
    `[${card.nameKo} (${card.nameEn}) — Fallback 참고 지식 (DB 직접 조회)]`,
    `⚠️ 주의: RAG 검색 품질이 낮아 데이터베이스에서 직접 가져온 정보입니다.`,
    // ... 도메인별 우선 정보
  ];
  return ragDocLines.join('\n');
}
```

**ChromaDB.py 원본:**
```python
if not results:
    return f"{selected_card}에 대한 구체적인 해석 정보를 찾을 수 없습니다."
```

**타로 마스터 개선:**
- ❌ 단순 에러 메시지 반환 → ✅ DB에서 직접 조회하여 기본 정보 제공
- ✅ Fallback 사실을 AI에게 명시 (할루시네이션 방지)

```json
    { "nameKo": "연인", "position": "현재", "isReversed": false },
    { "nameKo": "세계", "position": "미래", "isReversed": false }
  ]

```

### AI 답변 구조 (Retrieve.py 패턴 적용)

#### 1️⃣ 서론 (인사/분위기)
```
안녕하세요! 새로운 연애에 대한 기대와 설렘이 느껴지는 질문이네요.
3장의 카드가 당신의 연애 여정에 대해 어떤 이야기를 들려주는지 함께 살펴볼까요?
```

#### 2️⃣ 본론 (카드 해석 + 멀티 카드 흐름)

**과거 — 바보:**
```
[상세 해석 250자 이상...]
과거의 당신은 순수한 열정과 호기심으로 가득했습니다...
```

**현재 — 연인:**
```
[상세 해석 250자 이상...]
그 순수한 에너지가 지금 연인 카드로 결실을 맺고 있어요...
```

**미래 — 세계:**
```
[상세 해석 250자 이상...]
이 흐름은 궁극적으로 완성과 성취의 세계 카드로 이어집니다...
```

**전체 흐름 (overallInterpretation):**
```
자, 3장의 카드가 그리는 전체 그림을 보세요!

과거의 바보 카드가 보여준 순수한 열정과 새로운 시작의 에너지가
→ 현재 연인 카드의 운명적 만남으로 이어지고
→ 미래 세계 카드의 완벽한 완성으로 귀결됩니다.

마치 하나의 아름다운 드라마처럼, 당신의 연애 여정은
순수함에서 시작해 진정한 사랑을 만나고,
최종적으로 완성된 관계로 발전할 가능성을 보여줍니다.
```

#### 3️⃣ 결론 (조언 및 격려)
```
🔮 카드들이 전하는 핵심 메시지:
새로운 연애는 시작될 뿐만 아니라, 매우 긍정적이고 완성도 높은 관계로
발전할 가능성이 큽니다.

✨ 지금 당장 실천할 행동 조언 3가지:
1. 순수한 마음으로 새로운 만남에 열려 있으세요
2. 직관을 믿고 운명적 느낌이 드는 사람에게 다가가세요
3. 완벽함을 기대하기보다 자연스러운 흐름을 따라가세요

💌 마음에 새길 격려 메시지:
당신은 이미 충분히 준비되어 있습니다. 카드들이 보여주는 이 아름다운
여정을 믿고, 작은 한 걸음부터 시작해보세요!
```

## 스트리밍 준비 (향후)

### Gemini Streaming API
```typescript
// 향후 구현 예정 (현재는 JSON 모드 사용)
async interpretWithStreaming(request: InterpretRequest) {
  const result = await model.generateContentStream({
    contents: [{ role: 'user', parts: [{ text: userPrompt }] }],
    systemInstruction: this.systemPrompt
  });

  // 스트리밍 청크 반환
  for await (const chunk of result.stream) {
    const chunkText = chunk.text();
    yield chunkText;  // 클라이언트로 실시간 전송
  }
}
```

**장점:**
- ✅ 긴 답변을 한 글자씩 실시간으로 표시
- ✅ 사용자 지루함 방지
- ✅ 응답 시작 시간 단축 (TTFB ↓)

**현재 상태:**
- JSON 모드로 구조화된 응답 반환 (정확성 우선)
- 프론트엔드 준비되면 스트리밍 전환 가능

## 트러블슈팅

### 문제: AI가 여전히 단조롭게 답변
**원인:** System prompt를 무시하거나 이해 못함
**해결:**
1. `temperature` 확인 (현재 0.8, 적정)
2. System prompt를 더 강조: "반드시", "중요" 키워드 추가
3. User prompt에도 구조 명시

### 문제: 멀티 카드 흐름이 연결 안 됨
**원인:** AI가 각 카드를 독립적으로 해석
**해결:**
- User prompt에 흐름 예시 자동 생성 (이미 적용됨)
```typescript
예시 흐름: "과거 → 현재 → 미래"
→ 첫 카드가 보여준 에너지가 → 둘째 카드의 상황으로 이어지고...
```

### 문제: 할루시네이션 발생 (없는 내용 지어냄)
**원인:** RAG 데이터 부족 + AI가 자의적 해석
**해결:**
1. Fallback으로 DB 직접 조회 (이미 적용됨)
2. System prompt에 "데이터에 없으면 유보적 표현" 명시
3. 검색 품질 스코어 임계값 조정 (현재 0.5)
